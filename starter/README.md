# Starter files for PathFinder Blackathon build

These files were generated during the office hours session (2026-04-24) to save build time. Move them into your real project structure once you scaffold the repo.

## Files

- **`youtube.ts`** — YouTube Data API v3 wrapper with 3-key rotation, quota-aware retry, search + ranking. Costs 101 quota units per ranked search. → Move to `convex/lib/youtube.ts`.
- **`onet-convert.ts`** — One-time converter. Reads downloaded O*NET text files, produces `onet.json` bundled with your Convex deploy. Run with Bun: `bun starter/onet-convert.ts ./db_29_X_text ./starter/onet.json`.
- **`onet.ts`** — In-memory O*NET data loader + lookup helpers (`onetLookup`, `onetSkillsForOccupation`, `computeSkillDiff`). Zero network calls. → Move to `convex/lib/onet.ts`.
- **`onet-fuzzy.ts`** — LLM-powered semantic lookup fallback for careers not in O*NET ("TikTok influencer", "vibe coder", etc.). Tries static lookup first, falls back to Claude Haiku for unmapped queries. Returns SOC + reasoning for the UI. → Move to `convex/lib/onet-fuzzy.ts`.
- **`onet.json`** (generated by you) — bundled occupation × skill data. → Move to `convex/data/onet.json`.

## Project root files (already created)

- **`../.env.local`** — Environment variable template with all keys you need to fill in. Already covered by `.gitignore`. Fill in keys as you provision them.
- **`../.gitignore`** — Standard Next.js + Convex + Vercel gitignore. Critically: blocks `.env*` files.

## Build day order (from the design doc)

### BEFORE the clock starts (pre-build)

1. Download O*NET text database from https://www.onetcenter.org/database.html (text format zip, ~50MB).
2. Unzip into the project root or wherever convenient.
3. Run conversion: `bun starter/onet-convert.ts ./db_29_X_text ./starter/onet.json` (replace `X` with the version you downloaded).
4. Confirm output JSON is ~5-15MB and contains 900+ occupations.
5. Spin up 3 Google Cloud projects, enable YouTube Data API v3 on each, paste keys into `../.env.local`.
6. Set $50 hard spend cap on Anthropic API key, paste key into `.env.local`.
7. Write the bridge-skills sentence for "radio host → UX designer" you want the judge to see.

### When the clock starts (build day)

1. `git init && gh repo create pathfinder --private` (or your preferred remote)
2. `npx create-next-app@latest .` (App Router, TypeScript, Tailwind)
3. `npx convex@latest dev` (creates `convex/` directory, populates `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL` in `.env.local`)
4. Move `starter/youtube.ts` → `convex/lib/youtube.ts`
5. Move `starter/onet.ts` → `convex/lib/onet.ts`
6. Move `starter/onet.json` → `convex/data/onet.json` (and update the import path at top of `onet.ts` accordingly: `import onetData from "../data/onet.json"`)
7. Define schema in `convex/schema.ts` (see design doc § Convex schema)
8. Continue with build order step 4+ in the design doc.
